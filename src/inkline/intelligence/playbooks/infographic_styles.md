# Infographic Styles Playbook

> **Purpose**: Guide the DesignAdvisor in selecting and composing infographic
> visual formats beyond standard charts — for use in slides, reports, and
> standalone infographic documents.
>
> **Authority sources**: Venngage, Column Five Media, Piktochart, Visme,
> Canva Design School.

---

## 1. Infographic Type Taxonomy

### 1.1 Statistical Infographic

**What it is**: A data-rich visual that combines multiple chart types, icons,
large numbers ("big stats"), and short text to tell a quantitative story.

**When to use**:
- Presenting survey results or research findings.
- Summarising KPIs or performance data.
- Making a data-heavy topic digestible for a general audience.

**Structure**:
```
┌──────────────────────────────┐
│  HEADLINE / TITLE            │
├──────────────────────────────┤
│  Big stat #1    Big stat #2  │
│   ████████       ████████    │
├──────────────────────────────┤
│  Bar chart or  │  Icon grid  │
│  line chart    │  or donut   │
├──────────────────────────────┤
│  Key takeaway callout box    │
├──────────────────────────────┤
│  Source / footer             │
└──────────────────────────────┘
```

**Rules**:
- Lead with the most striking number in oversized type (48pt+).
- Limit to 3-5 data points per section — don't overload.
- Alternate between chart types to maintain visual variety.
- Use icons to make abstract numbers concrete (e.g., person icon for headcount).

### 1.2 Timeline / Process Flow

**What it is**: A linear or branching sequence showing events, steps, or
milestones in chronological or logical order.

**When to use**:
- Company history, project milestones, product roadmaps.
- Step-by-step processes ("how it works").
- Historical narratives.

**Layout options**:

| Layout | Best for |
|--------|----------|
| Horizontal timeline | 3-8 events, slide format |
| Vertical timeline | 5-20 events, report/scroll format |
| Serpentine (zigzag) | 10+ events, poster/infographic format |
| Circular timeline | Recurring/cyclical processes |

**Rules**:
- Use consistent spacing between events (proportional to time, or equal).
- Alternate content left/right on vertical timelines for readability.
- Highlight key moments with colour accents or larger nodes.
- Include dates/labels at every node.

---

### 1.3 Comparison Layout

**What it is**: A side-by-side or structured comparison of two or more options,
products, plans, or concepts.

**When to use**:
- "Option A vs. Option B" decisions.
- Pricing tier comparisons.
- Before/after analyses.
- Competitor benchmarking.

**Layout options**:

| Layout | Best for |
|--------|----------|
| Split screen (2-column) | Binary comparison (A vs B) |
| Multi-column cards | 3-4 options (pricing tiers) |
| Feature matrix (table) | Many attributes across many options |
| Venn diagram | Overlapping shared/unique features |

**Rules**:
- Keep the same metrics/attributes in the same position for each option.
- Use visual hierarchy: bold the "winner" or recommended option.
- Limit to 5-7 comparison attributes to avoid table fatigue.
- Use check/cross icons for binary attributes.

---

### 1.4 Icon Grid / Pictogram

**What it is**: A grid of repeated icons where each icon represents a unit
(e.g., 1 icon = 1,000 people). Filled vs. unfilled icons show proportions.

**When to use**:
- Making abstract statistics tangible ("7 in 10 people...").
- Showing proportions in a more engaging way than a pie chart.
- Audience is non-technical.

**Rules**:
- Each icon = a fixed, labelled unit. State it clearly ("Each figure = 100 employees").
- Use a 10×10 grid (100 icons) for percentage-based displays.
- Fill icons from left-to-right, top-to-bottom.
- Use at most 2-3 colours (filled vs. unfilled, plus one highlight).
- Works best for round numbers. Avoid fractions of icons.

---

### 1.5 Progress Bars / Gauges

**What it is**: Linear or radial indicators showing completion percentage or
progress toward a goal.

**When to use**:
- Project status dashboards.
- Goal tracking (fundraising, OKRs, milestones).
- Skill/proficiency ratings.

**Variants**:

| Variant | When to use |
|---------|-------------|
| Horizontal bar | Progress toward a target |
| Circular/radial | Single KPI, compact space |
| Step progress | Discrete stages (1/5 complete) |
| Thermometer | Fundraising, budget burn |

**Rules**:
- Always label the current value AND the target.
- Use colour to indicate status (green = on track, amber = at risk, red = behind).
- For step progress, clearly distinguish completed vs. remaining steps.

---

### 1.6 Hierarchy / Pyramid Diagram

**What it is**: A layered triangle or org-chart structure showing levels of
importance, authority, or quantity.

**When to use**:
- Maslow-style priority frameworks.
- Organisational structures.
- Segmentation (few high-value → many low-value).
- Conceptual frameworks with ranked layers.

**Rules**:
- Widest layer at the bottom = largest quantity or broadest category.
- Limit to 3-6 layers for readability.
- Label each layer with both the category name and a key metric.
- Use colour gradient (darkest at top or bottom) to reinforce hierarchy.

---

### 1.7 Flowchart / Decision Tree

**What it is**: A branching diagram that shows logic paths, decision points, or
process workflows.

**When to use**:
- Decision-making guides ("Should you do X?").
- Process documentation (approval workflows, system architecture).
- Troubleshooting guides.
- Algorithm explanation.

**Rules**:
- Use standard shapes: rectangles (process), diamonds (decision), rounded
  rectangles (start/end), parallelograms (I/O).
- Flow top-to-bottom or left-to-right (never mix directions).
- Label every arrow with the condition (Yes/No, or descriptive).
- Keep to one page — if the flow is too complex, break into sub-flows.

---

### 1.8 Geographic / Map Infographic

**What it is**: Data overlaid on a map to show spatial patterns, regional
comparisons, or location-specific information.

**When to use**:
- Regional performance data.
- Office/facility locations.
- Market presence or expansion plans.
- Demographic data by region.

**Variants**:

| Variant | Best for |
|---------|----------|
| Choropleth | Rate/ratio comparison across regions |
| Bubble map | Magnitude at specific locations |
| Pin/marker map | Discrete locations |
| Flow/connection map | Routes, trade, migration |

**Rules**:
- Use a sequential colour scale for choropleth (one hue, varying lightness).
- Size bubbles by area, not radius.
- Remove unnecessary geographic detail (simplified boundaries are fine).
- Always include a legend and scale.

---

### 1.9 Venn Diagram

**What it is**: Overlapping circles showing shared and unique attributes between
2-3 groups.

**When to use**:
- Showing what groups have in common vs. what is unique.
- Skill overlap analysis.
- Market segment overlap.

**Rules**:
- Limit to 2-3 circles (4+ becomes unreadable).
- Label each zone (unique A, unique B, overlap A∩B).
- Use semi-transparent fills so overlaps are visible.
- Don't use Venn diagrams for precise quantitative comparisons.

---

### 1.10 Word Cloud / Tag Cloud

**What it is**: A visual arrangement of words where size represents frequency or
importance.

**When to use**:
- Showing dominant themes in qualitative data (survey responses, social media).
- Visualising tag/keyword frequency.

**When NOT to use**:
- When precise comparison matters (word sizes are hard to compare accurately).
- For quantitative analysis.
- In formal reports or investor presentations (perceived as informal).

**Rules**:
- Remove stop words and normalise variants (e.g., "running" → "run").
- Limit to 30-50 words maximum.
- Use a single colour family or 2-3 colours max.
- Horizontal orientation is more readable than random angles.

## 2. Composition Principles for Infographics

### Layout Flow

Infographics should guide the eye in a clear, predictable path:

| Flow pattern | When to use |
|-------------|-------------|
| **Top-to-bottom** | Default for vertical infographics, reports |
| **Left-to-right** | Timelines, process flows, slides |
| **Z-pattern** | Web pages, dashboards |
| **F-pattern** | Text-heavy content (users scan left column first) |

### Visual Hierarchy (Sizing)

1. **Hero element** — the single most important number, chart, or image. Largest
   on the page.
2. **Supporting elements** — secondary charts, icons, sub-stats. Medium size.
3. **Context elements** — labels, source notes, legends. Smallest.

### Grouping

- Use **proximity** to group related items (Gestalt principle).
- Use **borders or background colour blocks** to separate distinct sections.
- Use **consistent spacing** — uneven spacing implies unintended relationships.

### Text in Infographics

- **Three font sizes maximum**: headline, sub-heading, body.
- Keep text blocks to 1-2 sentences — infographics are visual-first.
- Use sentence case for readability (not ALL CAPS except for very short labels).

## 3. Decision Table — Which Infographic Style?

| Content type | Recommended style |
|-------------|-------------------|
| Survey results / statistics | Statistical infographic |
| Company history / milestones | Timeline |
| Product comparison | Comparison layout |
| "X in Y people" proportion | Icon grid / pictogram |
| Goal / OKR tracking | Progress bars |
| Org structure / framework | Hierarchy / pyramid |
| Process / workflow | Flowchart |
| Regional data | Map infographic |
| Shared vs unique features | Venn diagram |
| Qualitative theme analysis | Word cloud |
| Step-by-step instructions | Process flow / numbered steps |

---

## 4. Anti-Patterns

| Anti-pattern | Problem | Fix |
|-------------|---------|-----|
| Decoration-heavy | Icons/images that don't encode data | Every visual element should communicate information |
| Text wall with icons | Just a document with clip art | Reduce text; let visuals carry the message |
| Too many fonts/colours | Visual chaos | Max 2 fonts, 3-5 colours |
| No visual hierarchy | Everything same size | Make the key insight 2-3x larger than supporting content |
| Missing source attribution | Undermines credibility | Always cite data sources |
| Inconsistent icon style | Mixing line/filled/3D icons | Use a single icon family throughout |
