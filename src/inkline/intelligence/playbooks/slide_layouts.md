# Slide Layout Patterns Playbook

> **Purpose**: Guide the DesignAdvisor in selecting professional slide layouts,
> structuring content using consulting-grade patterns, and applying the narrative
> frameworks used by McKinsey, BCG, and top design firms.
>
> **Authority sources**: McKinsey presentation guidelines, Barbara Minto's Pyramid
> Principle, Slideworks consulting guides, SlideModel, Beautiful.ai, think-cell.

---

## 1. The Anatomy of a Consulting Slide

Every professional slide has three structural zones:

```
┌──────────────────────────────────────────────┐
│  ACTION TITLE (1-2 lines)                    │  ← The insight / "so what"
│  ─────────────────────────────────────────    │
│                                              │
│              BODY / EXHIBIT                  │  ← Evidence supporting the title
│     (chart, table, diagram, or text)         │
│                                              │
│                                              │
├──────────────────────────────────────────────┤
│  Source: ... | Footer / page number          │  ← Provenance and navigation
└──────────────────────────────────────────────┘
```

### Action Titles (Not Descriptive Titles)

The single most important rule in consulting slide design:

| Bad (descriptive) | Good (action title) |
|-------------------|---------------------|
| "Revenue Overview" | "Revenue grew 18% YoY, driven by APAC expansion" |
| "Cost Breakdown" | "SG&A costs are 2x industry benchmark, signalling inefficiency" |
| "Customer Segments" | "Enterprise segment accounts for 72% of margin despite 31% of volume" |

**Rules for action titles**:
- State the insight, conclusion, or recommendation — not the topic.
- Should be readable as a standalone storyline (read all titles in sequence).
- Maximum 2 lines, 12-15 words.
- Use active voice and specific numbers.

## 2. Narrative Frameworks

### 2.1 The Pyramid Principle (Minto)

Barbara Minto's framework, standard at McKinsey:

```
            ┌─────────────┐
            │  ANSWER      │   ← Lead with the conclusion
            └──────┬───────┘
         ┌─────────┼─────────┐
    ┌────┴───┐ ┌───┴────┐ ┌──┴─────┐
    │ Point 1│ │ Point 2│ │ Point 3│   ← Supporting arguments (MECE)
    └────┬───┘ └───┬────┘ └──┬─────┘
    ┌──┴──┐   ┌──┴──┐   ┌──┴──┐
    │ Data │   │ Data │   │ Data │   ← Evidence
    └─────┘   └─────┘   └─────┘
```

**Key principles**:
- **Top-down delivery**: Answer first, then support. "You think bottom-up but present top-down."
- **MECE grouping**: Supporting points are Mutually Exclusive, Collectively Exhaustive.
- **"So what?" test**: Every slide must pass — "Why should the audience care?"
- **Govern with 3s**: Group arguments in threes (occasionally 2 or 4, never more than 5).

### 2.2 SCR Framework (Situation-Complication-Resolution)

Used for executive summaries and presentation storylines:

| Component | What it answers | Example |
|-----------|----------------|---------|
| **Situation** | "What is the context?" | "The company has grown revenue 15% annually for 5 years" |
| **Complication** | "What changed / what's the problem?" | "Margins have eroded from 28% to 19% due to rising input costs" |
| **Resolution** | "What should we do?" | "Implement a 3-lever cost reduction programme targeting $40M savings" |

**Rules**:
- The exec summary slide(s) should follow SCR in sequence.
- Each body section then elaborates on one element of the resolution.
- The resolution must be specific and actionable, not vague.

### 2.3 SCQA Variant (Situation-Complication-Question-Answer)

Similar to SCR but inserts an explicit question before the answer:

- Situation → Complication → "How can we restore margins?" → Answer
- Useful when the audience needs to be led to the question before hearing the answer.

## 3. Slide Layout Library

### 3.1 Title Slide

```
┌──────────────────────────────────────┐
│                                      │
│         PRESENTATION TITLE           │
│         Subtitle / context           │
│                                      │
│         Author | Date | Confidential │
└──────────────────────────────────────┘
```

**Rules**: Clean, minimal. Company logo top-right or bottom-left. No charts or data.

---

### 3.2 Section Divider

```
┌──────────────────────────────────────┐
│                                      │
│   ■ SECTION NUMBER                   │
│     Section Title                    │
│                                      │
└──────────────────────────────────────┘
```

**Rules**: Full-colour background or large type. No body content. Acts as a visual "chapter break."

---

### 3.3 Single Chart + Action Title

```
┌──────────────────────────────────────┐
│  Action title stating the insight     │
│  ──────────────────────────────       │
│  ┌────────────────────────────┐      │
│  │                            │      │
│  │         CHART              │      │
│  │                            │      │
│  └────────────────────────────┘      │
│  Source: ...                          │
└──────────────────────────────────────┘
```

**When to use**: One chart tells the whole story. The most common consulting slide.

---

### 3.4 Two-Chart Comparison

```
┌──────────────────────────────────────┐
│  Action title comparing two findings  │
│  ──────────────────────────────       │
│  ┌──────────┐    ┌──────────┐        │
│  │ CHART A   │    │ CHART B   │        │
│  │           │    │           │        │
│  └──────────┘    └──────────┘        │
│  Source: ...                          │
└──────────────────────────────────────┘
```

**When to use**: Showing a before/after, two metrics, or two segments side by side.

---

### 3.5 2x2 Matrix

```
┌──────────────────────────────────────┐
│  Action title                         │
│         High ▲                        │
│    ┌─────────┬─────────┐             │
│    │ Q2      │ Q1       │             │
│    │ Niche   │ Stars    │             │
│  ──┼─────────┼─────────┤──► High     │
│    │ Q3      │ Q4       │             │
│    │ Dogs    │ Cash Cows│             │
│    └─────────┴─────────┘             │
│  Source: ...                          │
└──────────────────────────────────────┘
```

**When to use**: Strategic frameworks — BCG matrix, Eisenhower matrix, prioritisation, risk assessment.

**Rules**:
- Label both axes clearly with the dimension being measured.
- Label each quadrant with a memorable name.
- Plot entities as dots/bubbles within quadrants if applicable.

---

### 3.6 Three-Column Cards

```
┌──────────────────────────────────────┐
│  Action title                         │
│  ┌──────┐  ┌──────┐  ┌──────┐       │
│  │ ICON │  │ ICON │  │ ICON │       │
│  │ Head │  │ Head │  │ Head │       │
│  │ Body │  │ Body │  │ Body │       │
│  │ text │  │ text │  │ text │       │
│  └──────┘  └──────┘  └──────┘       │
└──────────────────────────────────────┘
```

**When to use**: Presenting 3 options, pillars, recommendations, or key findings in parallel.

**Rules**:
- Each card must have the same structure (icon, heading, body).
- Limit body text to 2-3 lines per card.
- Works with 2, 3, or 4 columns (never more than 4 on one slide).

---

### 3.7 KPI Dashboard Slide

```
┌──────────────────────────────────────┐
│  Action title                         │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐       │
│  │ KPI│ │ KPI│ │ KPI│ │ KPI│       │
│  │ $4M│ │ 23%│ │ 150│ │ A+ │       │
│  │ ▲12%│ │ ▼3%│ │ ═  │ │ ▲  │       │
│  └────┘ └────┘ └────┘ └────┘       │
│  ┌─────────────────────────────┐     │
│  │      SUPPORTING CHART       │     │
│  └─────────────────────────────┘     │
│  Source: ...                          │
└──────────────────────────────────────┘
```

**When to use**: Status updates, board reports, monthly reviews.

**Rules**:
- 3-5 KPI cards across the top, each with: metric name, value, trend indicator.
- Use RAG colours (red/amber/green) for status.
- One supporting chart below for context.

---

### 3.8 Split Layout (Image + Text)

```
┌──────────────────────────────────────┐
│  ┌──────────────┐ ┌──────────────┐  │
│  │               │ │              │  │
│  │    IMAGE      │ │   HEADLINE   │  │
│  │    or         │ │   Body text  │  │
│  │    PHOTO      │ │   • Bullet 1 │  │
│  │               │ │   • Bullet 2 │  │
│  │               │ │   • Bullet 3 │  │
│  └──────────────┘ └──────────────┘  │
└──────────────────────────────────────┘
```

**When to use**: Case studies, qualitative insights, team bios, product features.

**Rules**:
- Image takes 40-50% of slide width; text takes the remainder.
- Image should be relevant (not decorative stock photos).
- Keep text side to 3-5 bullet points maximum.

### 3.9 Full-Bleed Image with Overlay

```
┌──────────────────────────────────────┐
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│ ░░░░  FULL IMAGE BACKGROUND  ░░░░░░ │
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│ ┌──────────────────────────┐ ░░░░░░ │
│ │  HEADLINE                │ ░░░░░░ │
│ │  Short supporting text   │ ░░░░░░ │
│ └──────────────────────────┘ ░░░░░░ │
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
└──────────────────────────────────────┘
```

**When to use**: Opening/closing slides, vision statements, emotional emphasis.

**Rules**:
- Use a semi-transparent overlay (60-80% opacity black/dark) behind text to ensure readability.
- Text should be minimal (1 headline + 1 sentence max).
- Image must be high resolution (min 1920x1080).

---

### 3.10 Bullet List / Text Slide

```
┌──────────────────────────────────────┐
│  Action title                         │
│  ──────────────────────────────       │
│                                      │
│  ■ Main point one with evidence       │
│                                      │
│  ■ Main point two with evidence       │
│                                      │
│  ■ Main point three with evidence     │
│                                      │
└──────────────────────────────────────┘
```

**When to use**: Recommendations, key findings, agenda slides.

**Rules**:
- Maximum 3-5 bullets per slide.
- Each bullet is a complete sentence (not a fragment).
- Use parallel grammatical structure ("Increase...", "Reduce...", "Invest...").
- DO NOT read bullets verbatim during presentation.
- Numbers over bullets when sequence matters.

---

### 3.11 Recommendation / Next Steps

```
┌──────────────────────────────────────┐
│  Recommendation: [Action title]       │
│  ──────────────────────────────       │
│  ┌─────────────────────────────────┐ │
│  │ # │ Action       │ Owner │ Date │ │
│  │ 1 │ Launch X     │ VP Ops│ Q3   │ │
│  │ 2 │ Hire Y team  │ CHRO  │ Q3   │ │
│  │ 3 │ Invest $Zm   │ CFO   │ Q4   │ │
│  └─────────────────────────────────┘ │
│                                      │
│  Expected impact: $XM savings by Q1  │
└──────────────────────────────────────┘
```

**Rules**:
- Each recommendation starts with an action verb.
- Include owner and timeline for every item.
- State expected impact quantitatively.

## 4. Slide Sequencing Patterns

### The "Ghost Deck" Method (McKinsey)

Before building slides, write only the action titles in sequence. If the titles
read as a coherent story, the structure is sound.

```
Slide 1:  "Revenue growth has stalled at 3% despite $50M investment"
Slide 2:  "Three root causes explain the stagnation"
Slide 3:  "Customer churn doubled in the mid-market segment"
Slide 4:  "Product-market fit is weakening in EMEA"
Slide 5:  "Pricing has not kept pace with value delivered"
Slide 6:  "A three-lever plan can restore 12% growth within 18 months"
...
```

### Standard Presentation Arc

| Section | Slides | Purpose |
|---------|--------|---------|
| Title | 1 | Set the stage |
| Executive summary | 1-2 | SCR: conclusion first |
| Context / situation | 2-4 | Shared understanding of the problem |
| Analysis / findings | 5-15 | Evidence organized by argument |
| Recommendations | 2-4 | Specific, actionable next steps |
| Appendix | As needed | Supporting detail for Q&A |

### Slide Count Guidelines

| Presentation type | Target slides | Time per slide |
|-------------------|---------------|----------------|
| Board meeting | 10-15 | 2-3 min |
| Investor pitch | 10-12 | 1-2 min |
| Strategy review | 20-30 | 1.5-2 min |
| Workshop read-out | 30-50 | 1 min |
| Leave-behind document | Unlimited | Read at own pace |

## 5. Formatting Standards

### Text

| Element | Size | Weight |
|---------|------|--------|
| Action title | 18-22pt | Bold |
| Sub-heading | 14-16pt | Bold or Semi-bold |
| Body text / bullets | 12-14pt | Regular |
| Chart labels | 10-12pt | Regular |
| Source / footer | 8-10pt | Light or Regular |

### Alignment

- All slide elements must align to a consistent grid.
- Left-align text (never justify; centre only for titles on special slides).
- Charts should occupy the same position and size across similar slides.

### Colour Usage

- **Primary colour**: Used for key data, headers, emphasis.
- **Grey palette**: Used for supporting elements, gridlines, secondary data.
- **Accent colour**: Used sparingly for callouts, highlights, one "hero" data
  point per slide.
- Limit to 3-5 colours per slide.

### Animations

- **Avoid** in professional contexts. Animations slow down navigation,
  complicate PDF export, and frustrate experienced audiences.
- Exception: progressive reveal in live presentations when managing audience
  attention on complex data.

## 6. Data-Heavy vs. Narrative Slides

| Characteristic | Data slide | Narrative slide |
|---------------|-----------|----------------|
| Primary element | Chart, table, or exhibit | Text, images, icons |
| Title style | Quantitative insight | Qualitative statement |
| Density | High — one complex exhibit | Low — white space, few elements |
| Audience | Analysts, deep-divers | Executives, broad audience |
| Example | "CAC increased 34% YoY..." | "We must pivot our acquisition strategy" |

**Rule**: A good deck alternates between data and narrative slides to maintain
rhythm and prevent fatigue.

---

## 7. Anti-Patterns

| Anti-pattern | Problem | Fix |
|-------------|---------|-----|
| "Slideument" | Too much text — neither good slide nor good document | Write a separate document; simplify the slide |
| Descriptive titles | "Revenue" instead of insight | Always use action titles |
| Wall of bullets | 8+ bullets with sub-bullets | Break into multiple slides or use a table |
| Logo soup | Excessive branding on every slide | Logo on title + footer only |
| Chart junk | Gridlines, borders, shadows, 3D | Remove all non-data ink |
| Inconsistent formatting | Different fonts, sizes, positions per slide | Use a master template |
| Reading the slide | Presenter reads bullets verbatim | Slides support the speaker, they don't replace them |
